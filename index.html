About Me

I'm a student looking to pursue a career in joint cybersecurity and software engineering endeavours in the future. With certificates in C and C++ software development from a joint Dartmouth University and Institut Mines-Télécom initiative and LinkedIn Learning, respectively, I am also actively looking to learn more about new and interesting technologies (and the ways that they can be broken).

"Michael has been a thorough and dedicated hacker and I enjoy receiving reports from him."

- VDP manager of an asset with over 500,000,000 users.

Over the last few years, I have discovered and disclosed a number of software security vulnerabilities in a combination of open/closed-source software - these vulnerabilities have ranged from buffer overflows in a kernel-mode driver to a trivial race condition in ticketing software, to full authentication bypasses in a widely used financial platform.

Increasingly, however, most of my focus has been towards reverse-engineering applications without the intent of discovering security flaws in them, allowing a shift towards more binary-centred research that has allowed me to develop a keen interest in reversing Windows drivers and desktop applications in general. This research was conducted with the use of tools such as x64dbg, Ghidra, IDA Pro, Rizin Cutter (formerly radare2, I believe), and some attempts at using WinDbg (though I was unable to get everything working smoothly in a virtual machine).

More recently, however, I have been developing a proxy system that makes use of post-quantum cryptographic primatives (namely, 'Kyber' as proposed by the CRYSTALS group to a NIST PQC project) and other interesting techniques to establish a degree of anonymity and message security over a peer-to-peer network.

Projects

I'm extremely interested in C++, C (especially kernel-mode programming on Windows), C#, Python, and reverse engineering projects in addition to the odd bit of web development (HTML, CSS, and JavaScript) when it seems necessary (like this website). Some of my interests have led to the development of personal projects to either solve issues or to teach me more about their fundamentals - quite a few of those projects are now open-source and are available via GitHub:

T2-Lib

A performant, secure C++ library which leverages the boost.asio framework to deliver an object-oriented networking model that can easily be used by other programs. This project was written in C++ (17/20) and is actively being used in another personal project.

AQArm

A simplistic UI-driven interpreter of a limited ARM subset. Written in C++ using the Qt Framework and designed with future modularity in mind, making additions to the instruction-set much simpler.

SiteMon

A Windows tool for monitoring websites for source-level changes in content, this project is configurable via regular expressions, timeouts, custom user-agents, and exportable (encrypted) configuration files. Written in C# using the .NET framework and WinForms for the user-interface.

PNG-CMD

A command-line application that allows users to explore the internals of PNG image files and their comprising chunks. Written in modern C and thoroughly tested using fuzzing (AFL-Fuzz) and a code assurance platform (Codacy).

PE-Imports

A Python script that enumerates over files or directories, printing the imports of all executables that it finds. This script is currently configured to only search for Windows drivers (using a glob query) but can easily be modified. A good use-case for this script is to have a large directory of interesting drivers which the script would then scan for interesting API imports (which depending on your classification of 'interesting', might range from ExAllocatePool to DbgPrintEx).

Proxy Project

An (in-development) peer-to-peer application-level proxy that attempts to deliver long-term confidentiality of messages in the event of relay nodes being compromised. Written in C++ (20) using boost and T2-lib with a C# user-interface.
The source code for this project is not available (send me an email if you'd like a copy of it).

Antivirus Project

A kernel-mode antivirus driver which interacts with a user-interface to detect malicious processes at runtime using heuristic and static detection functionality. The driver was written in C using the Kernel-Mode Driver Framework (KMDF) in addition to a user-interface which was written in C++ using the Qt Framework.
The source code for this project is not available (send me an email if you'd like a copy of it).
