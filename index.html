Projects

I'm extremely interested in C++, C (especially kernel-mode programming on Windows), C#, Python, and reverse engineering projects in addition to the odd bit of web development (HTML, CSS, and JavaScript) when it seems necessary (like this website). Some of my interests have led to the development of personal projects to either solve issues or to teach me more about their fundamentals - quite a few of those projects are now open-source and are available via GitHub:

T2-Lib

A performant, secure C++ library which leverages the boost.asio framework to deliver an object-oriented networking model that can easily be used by other programs. This project was written in C++ (17/20) and is actively being used in another personal project.

AQArm

A simplistic UI-driven interpreter of a limited ARM subset. Written in C++ using the Qt Framework and designed with future modularity in mind, making additions to the instruction-set much simpler.

SiteMon

A Windows tool for monitoring websites for source-level changes in content, this project is configurable via regular expressions, timeouts, custom user-agents, and exportable (encrypted) configuration files. Written in C# using the .NET framework and WinForms for the user-interface.

PNG-CMD

A command-line application that allows users to explore the internals of PNG image files and their comprising chunks. Written in modern C and thoroughly tested using fuzzing (AFL-Fuzz) and a code assurance platform (Codacy).

PE-Imports

A Python script that enumerates over files or directories, printing the imports of all executables that it finds. A good use-case for this script is to have a large directory of drivers which the script would then scan for interesting API imports (might range from ExAllocatePool to DbgPrintEx).

Proxy Project

A peer-to-peer proxy that uses post-quantum cryptography to secure data. Written in C++ (20) using boost and T2-lib with a C# user-interface.

Antivirus Project

A kernel-mode antivirus driver which interacts with a user-interface to detect malicious processes at runtime using heuristic and static detection functionality. The driver was written in C using the Kernel-Mode Driver Framework (KMDF) in addition to a user-interface which was written in C++ using the Qt Framework.
